# A/B Test: Survey Before Telegram Redirect

## Overview
Implement an A/B test on the landing page. When a user loads the page, they are randomly assigned to variant A (control) or variant B (test). The assignment must be consistent per session.

## Variant A — Control
No changes. User clicks purchase button → redirects to Telegram bot (current behavior).

## Variant B — Survey
User clicks purchase button → shown a 4-question survey → after completing → redirects to Telegram bot.

The existing `script.js` logic (attribution, ID generation, Supabase insert) stays unchanged. The survey results are saved to a **new Supabase table** using the same attribution ID already generated by `script.js`.

## Survey Questions (Uzbek)

### 1. Qanday turdagi kiyim sotasiz?
- Ayollar kiyimi
- Erkaklar kiyimi
- Bolalar kiyimi
- Hammasi
- Kiyim emas, boshqa narsa sotaman

### 2. Hozir mahsulotlaringizni qanday suratga olasiz?
- Telefonimda o'zim suratga olaman
- Fotograf yollayman
- Internetdagi suratni joylayman
- Hali yaxshi suratlarim yo'q

### 3. Mahsulot suratga olishda sizni eng ko'p nima bezovta qiladi?
- Juda qimmat
- Juda uzoq vaqt oladi
- Sifati yetarli emas
- Model topish qiyin

### 4. Mahsulotlaringizni qayerda sotasiz? *(bir nechta tanlash mumkin)*
- Instagram
- Telegram
- Marketplace (Uzum/OLX)
- O'z veb-saytim
- Faqat do'konda

## Technical Requirements
- Random 50/50 split on page load (A or B)
- Store variant assignment in sessionStorage for consistency
- New Supabase table `survey_responses` with: attribution ID, variant, answers, timestamp
- Log the assigned variant in `attribution_logs` (new column or in the new table)
- Survey UI must match existing landing page design (dark theme, CSS variables, glassmorphism)
- ES5-compatible JavaScript (var, function keyword, no arrow functions)
- Survey appears as overlay/modal or inline step after CTA click (for variant B only)
- After survey submission → redirect to Telegram bot as normal
